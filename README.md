# PoE Timeless Jewels

流亡黯道（Path of Exile）永恆珠寶查詢與計算工具。

## 主要功能

- 依珠寶類型、種子（Seed）與條件篩選結果
- 顯示對應詞綴與天賦樹影響
- 支援 JSON 匯出與匯入
- 前端以 WASM 計算，提供快速查詢體驗

## 版本與更新紀錄

### 2026-04-14（v1.5.3）

- 版本更新至 `v1.5.3`。
- 珠寶、征服者與聯盟選項名稱改為跟隨目前語系顯示，不再只有繁體中文與英文。
- 收藏珠寶卡片與收藏編輯器改為即時依語系顯示珠寶與征服者名稱，避免沿用收藏當下保存的舊 label。
- 台服交易查詢仍固定使用繁體中文聯盟名稱送出，避免多語系 UI 影響交易連結。
- 首頁標題影片新增簡體中文、英文與日文版本，會隨目前語系切換。

### 2026-04-14（v1.5.2）

- 版本更新至 `v1.5.2`。
- UI 介面文案擴充為 11 語，現在會隨語系切換同步顯示 `tw`、`en`、`cn`、`de`、`es`、`fr`、`jp`、`kr`、`po`、`ru`、`th`。
- 詞綴與 UI 現在共用同一套官方語系列表，瀏覽器自動偵測與手動切換都會一起生效。
- 多語系用語對照以 PoEDB 的永恆珠寶頁面為主要參考來源之一，避免珠寶、征服者、反查等專有名詞在不同語系下前後不一致。

### 2026-04-14（v1.5.1）

- 版本更新至 `v1.5.1`。
- 修正本機預覽與靜態建置流程，現在 `pnpm build` 會先以 `GOOS=js`、`GOARCH=wasm` 重新編譯 `calculator.wasm`，避免誤將 Windows 可執行檔當成 WASM 載入。
- `open-local-preview.ps1` 新增 WASM 檔頭檢查；若偵測到 `build/calculator.wasm` 不是有效 WASM，會自動改用重新建置後再啟動預覽。
- 同步更新 `wasm_exec.js` 複本，降低本機 Go 版本更新後前後端執行時不一致的風險。
- 修正多語系切換時的反應性問題；搜尋結果卡片與天賦樹提示現在會隨語系切換即時刷新，不再只剩繁體中文與英文看起來有生效。

### 2026-04-13（v1.5.0）

- 版本更新至 `v1.5.0`。
- 資料基準切換至 `3.28`，同步更新永恆珠寶原始資料、技能樹資料與詞綴翻譯來源。
- 新增官方詞綴多語系支援，接入 `go-pob-data-main/data/3.28/stat_translations` 的全部官方語系：
  `tw`、`en`、`cn`、`de`、`es`、`fr`、`jp`、`kr`、`po`、`ru`、`th`。
- 新增語系自動偵測與手動切換；會優先依瀏覽器語系選擇，並將使用者選擇保存到本機。
- 英文模式下，詞綴改為純英文顯示，不再額外附帶雙語；其他官方語系維持「當前語系 / 英文」對照。
- 首頁、載入畫面、查詢面板、反查結果與收藏面板已接上語系切換；非繁體中文介面目前以英文 UI 為主。
- `update_assets.ps1` 已更新為同步下載全部官方詞綴語系，避免之後更新資產時遺失多語系資料。

### 2026-04-09（v1.4.0）

- 版本更新至 `v1.4.0`。
- 收藏珠寶新增「回推查詢」操作，可將收藏內容帶回天賦樹查詢面板。
- 新建立的收藏會同步保存查詢上下文，之後可完整回填珠寶、征服者、插槽位置、反查詞綴與權重條件。
- 舊的單顆收藏若未保存查詢上下文，會嘗試依收藏快照自動推回插槽位置。
- 舊的群組收藏若缺少原始查詢條件，會先帶入可還原的珠寶與征服者資訊，並提示需重新收藏才能完整回推。
